 # Linux에 압축본 JDK 설치하기 ( wget, tar )

JDK 9 다운로드.
wget : 웹 브라우저를 통해 어떠한 파일을 다운로드함.
tar -zxvf 파일이름 : 파일 압축풀기. 

설치 확인.
cd jdk디렉토리/bin -> ./java -version -> 정상적으로 설치되면 자바 버전이 뜬다.



 # 링크파일 사용하기 ( ln, path, cp )

링크 파일.
ls -l을 했을 때 링크파일은 lrwx.... 으로 첫 시작을 l로한다.
실제 실행파일을 좀 더 간편하게 사용하기 위한 바로가기 파일을 의미한다.

링크 파일 종류.
Symbolic Link : ln -s를 통해 만든다. ( 원본파일을 가르키는 파일 )
Hard Link : ln을 통해 만든다. ( 원본파일에 별칭을 만드는 느낌의 파일 - 많이 사용하지 않는다 )

링크 파일 생성.
ln -s 원본파일이름 링크파일이름 ;



 # 링크 파일 실습.

Hard Link.
Hard Link의 내용을 수정하면 원본 파일도 수정된다.
Hard Link의 원본을 삭제해도 Hard Link는 남겨진다.

 # 링크파일을 이용한 실행파일 리졸빙.

bin 디렉토리.
실행파일을 저장하는 디렉토리.
환경변수 PATH에 bin 디렉토리는 기본값으로 설정되어 있다.

실행파일의 링크파일을 bin 디렉토리에 생성하기.
1. cd bin.
2. ln -s 실행파일경로 링크파일이름 ;



 # 링크파일로 컴파일과 실행하기.

자바 관습.
파일이름 ( 파일이름.java )와 class 이름을 똑같이 맞춰야된다.

자바파일 컴파일하기.
javac 파일이름.java -> 클래스이름.class가 생성.

컴파일된 클래스파일 실행하기.
java 클래스이름 -> 실행.



 # 사용자 추가하기 useradd, usermod, userdel.

사용자 관련 명령어.
useradd : 사용자 추가.
usermod : 사용자 변경.
userdel : 사용자 삭제.

사용자 추가하기.
useradd 사용자이름.

사용자 패스워드 추가.
sudo passwd 사용자이름 

사용자 추가 확인하기.
cat /etc/passwd

터미널 변경.
alt + F1 ~ F6.



 # 사용자 추가하기 실습.

추가된 사용자.
HOME 디렉토리가 없다.

계정 추가하기.
sudo useradd 사용자이름 ;

계정 비밀번호 설정.
sudo passwd 사용자이름 ;
비밀번호 입력 ;

HOME 디렉토리 추가.
sudo mkdir /home/사용자이름 ;

그룹 확인하기.
groups 사용자이름 -> 사용자의 그룹을 확인.



 # 홈 디렉토리 생성 소유권 변경.

소유권 변경.
sudo chown 사용자이름 파일이름

소유권 그룹 변경.
sudo chown 사용자이름:그룹이름 파일이름

소유권 확인.
touch 파일이름 ( 소유권을 가진 디렉토리에서 파일을 생성해본다 )



 # 사용자를 추가하는 스크립트 파일 생성하기.

설명.
사용자를 생성할때마다 여러 설정들을 매번 다시 작성하기 힘들기 때문에 스크립트 파일을 만들어놓는다.

파일 생성.
nana 파일이름.
useradd 유저이름
tail -n2 /etc/passwd
mkdir /home/유저이름
chown 유저이름:유저그룹이름 /home/유저이름
echo "user added"



 # 파일권한 변경.

파일 실행 권한.
ls -l ( 실행 권한 확인 )
-rwxrwxrwx : 파일타입 / 소유자 / 소유그룹 / 다른사용자 
r read : 파일을 읽을 수 있는 권한.
w write : 파일을 작성 및 수정할 수 있는 권한.
x excute : 파일을 실행할 수 있는 권한.

파일 권한 설정.
u = 유저.
g = 그룹.
o = 다른사용자.
a = 모든유저.
-s, +s = 관리자권한.
chmod 644 파일이름 ;
chmod u+s 파일이름 ;
chmod u=w 파일이름 ;
chmod go+r 파일이름 ;
chmod o -rwx 파일이름 ;

파일 위치 찾기.
whereis 파일이름 ;


 
 # 인자를 이용한 스크립트 파일로 변경하기.

스크립트 파일에 매개변수 설정.
$1 : 첫 번째 매개변수.
$2 : 두 번째 매개변수.

스크립트 파일 실행.
파일이름 매개변수1 매개변수2 ... ;

환경변수.
PATH에 순서에 따라 파일이름이 동일해도 먼저 실행되는 파일이 존재한다.
which 파일이름 : 현재 위치에서 실행되는 파일이름을 알려준다. ( sudo which 파일이름이 다를 수 있다 )

관리자 권한.
sudo su - 
exit : 관리자 권한 로그아웃.
